Setting up Code Composer Studio IDE


Code Composer Studio

Code Composer Studio is an integrated development environment (IDE) that supports TI's Micro-controller and Embedded Processors portfolio. Code Composer Studio comprises a suite of tools used to develop and debug embedded applications.

We recommend to use the CCS IDE for the TI target devices i.e CC26x2r1 especially for debugging.

Install CCS and Compiler

The Code Composer Studio is available in Windows (64-Bit only), Linux and MAC OS platforms.

  • Download and install Code Composer Studio.
  • Download and install the GCC ARM none-eabi compiler.
  • It is necessary to have make installed at your system. If you do not have it, you can install e.g. MinGW.
  • It is also necessary to have Git installed at your system. If you do not have it, you can install e.g. Git.

ATTENTION: For Windows it is necessary to have the install path of gcc-arm-none-eabi Git and make at the Windows environment path variable. Make sure to add these paths:

  • C:/Program Files (x86)/GNU Tools Arm Embedded/your-version-q4-major/bin;
  • C:/Program Files (x86)/mingw32/bin;
  • C:/Program Files/Git/bin
  • C:/Program Files/Git/usr/bin

Import project

Once you have started to launch the CCS, it will ask you to select a workspace. You can either select an empty workspace or an existing one to start working on the Stack project.

CCS start page
  • Import CCS project
    • Select the project-directory
    • Browse
    • Navigate to the root path of the project directory.
    • Select the project folder.
    • Finish
    • Do not enable XDCtools for the imported project and answer the related queries with 'No'
Import a CCS project

Setup Project

  • Setup CCS project
    • Select the project
    • Click Properties
    • Build
      • Unselect Use default build command and fill in make or `mingw32-make (it depends to your system).
      • Unselect Generate Makefiles automatically and fill in at Build directory the path of the application directory of the project (e.g. ...\app\app-sc).
      • Fill in the application name at Build (incremental build) in the Make build targets section (e.g. app-sc)
      • Apply and Close
CCS Project setup

Now the project is completely configured.

Build Project

  • Right-click on the project on the Project Explorer tab and select Build Project

If everything was successful the firmware binary file will be placed at the application folder. The file will have the suffix simplelink (e.g. app-sc.simplelink).

Debug Project

  • Select Run>Debug or press F11

The easiest way to get a CCS debug configuration is to simple click at the debug symbol. CCS will automatically create a new debug configuration. After it CCS will start the debugger also. If you have not set the program beforehand you will get an error. If this is the case, please open the debug configuration and go to the Program tab. At the section Program you can select the path to the binary file. The binary firmware file will be copied into the app folder and has the suffix simplelink (e.g. app-sc.simplelink).
